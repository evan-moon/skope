skope는 기본적으로 **로컬 우선**(local-first)이고 **키가 필요 없습니다**. 프로필과 기사 원장은 `~/.skope/skope.db` 단일 SQLite 파일에 저장됩니다. Claude가 당신을 위해 웹을 검색하는 경우를 제외하면, 어떤 것도 당신의 기기를 떠나지 않습니다.

## 설치

```bash
npm install -g @evan-moon/skope
```

이것으로 `skope` CLI(안전망)와 번들된 `skope-mcp` 서버(분석이 사는 곳)가 설치됩니다.

## Claude에 연결

```bash
skope mcp install
```

**Claude Desktop**과 **Claude Code** 양쪽 설정 파일에 MCP 서버 항목을 기록해 skope를 등록합니다. 활성화하려면 Claude Desktop을 재시작하십시오. Claude Code는 다음 세션에서 인식합니다.

## Claude Desktop 수동 연결

설정을 직접 편집하고 싶다면 `~/Library/Application Support/Claude/claude_desktop_config.json`에 추가합니다.

```json
{
  "mcpServers": {
    "skope": {
      "command": "node",
      "args": ["<번들된 skope-mcp.js 경로>"]
    }
  }
}
```

`skope mcp install`이 그 경로를 대신 찾아주므로, 위 명령이 권장 경로입니다.

## 프로필 만들기

skope는 무언가를 스코어링하기 전에 관심 프로필이 필요합니다. 가장 빠른 길은 Claude에게 만들게 하는 것입니다. 그냥 요청하면 Claude가 [`update_profile`](/ko/docs/mcp-tools#update_profile)을 호출하고, 필요하면 firma(자산)나 memex(메모리)에서 페더레이션해 옵니다.

LLM 없이 콜드 스타트하려면 CLI가 위치로부터 하나를 시드합니다.

```bash
skope init --location "Seoul, Korea" --languages ko,en
```

전체 명령은 [CLI 레퍼런스](/ko/docs/cli)를 참고하십시오.

## 무엇이 생성되나

```
~/.skope/
  skope.db, 프로필, 기사 원장, 읽음 로그 (SQLite)
  config.json, scan_news용 선택적 Tavily 키
```

## 선택: Tavily 키

skope의 주 수집 경로([`ingest_news`](/ko/docs/mcp-tools#ingest_news))는 **키가 필요 없습니다.** Claude가 직접 웹을 검색합니다. [Tavily](https://tavily.com) 키는 자체 웹 검색이 없는 클라이언트가 쓰는 [`scan_news`](/ko/docs/mcp-tools#scan_news) 폴백에만 필요합니다.

```bash
skope config set tavily-key tvly-xxxxxxxx
```
